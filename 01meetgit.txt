问题一：集中式VS分布式版本控制系统。
【集中式】版本库存放在中央服务器，集中式版本控制就是，先在自己本地进行修改，然后将修改提交给服务器。
类似于：中央服务器是图书馆，你要改一本书，先从图书馆借来，本地修改，再将图书归还。
集中式版本控制最大的问题是：必须联网才能工作，遇到网速不好，则会出现问题。
【分布式】不存在“中央服务器”，每个人电脑上都有完整的版本库，工作时不需要联网。多人协作时，将各自修改推送给对方则可以实现协作。
分布式的好处是：安全性较高，某一个人的版本库出现问题可以从其他人那里获取，但是中央服务器坏了的话，则所有人均不能工作。
实际操作中，为了实现共享“修改”的方便，创建一个服务器代理，大家将所有的修改都传给这个代理的服务器。

问题二：创建版本库
版本库又名仓库，英文名repository，可以理解为一个目录，目录里面所有文件都可以被GIT管理，每个文件的修改，删除，GIT都能跟踪，以便可以追踪历史，或者在将来某个时刻可以还原。
#Linux版本安装#
第一步创建版本库存放的目录
$ mkdir learngit #创建目录
$ cd learngit    #切换到目录
$ pwd            #显示整个目录名
/users/.../learngit
第二步安装git版本库在创建好的目录中。
$ git init
Initialized empty Git repository in /users/.../learngit/.git/
